非关系型数据库和非关系型数据库

关系型数据库

  • 遵循关系型数据库
    • 固定的表结构，数据结构改动麻烦
  • 维护事物一致性
  • 支持海量数据的吃鸡存储
  • 可用性不高
    • 数据库系统升级需要停机维护
      例：Oracle 、 DB2 、 MySQL等等。这些都是关系型数据库

非关系型数据库

  • 易于扩展
    • 不遵循关系型数据库模型，无需预先定义表的结构
  • 支持分布式数据库结构
  • 能够支持大量的零散的数据
    -数据库结构简单，读写性能好
  • 重点关注性能，而不是事物的一致性
    例：MongoDB、Apache CouchDB、ArangoDB等等。都是非关系型数据库

非关系型数据库的分类

非关系型数据库的分类，分为4类：

  1. 基于列的数据库
  2. 图形数据库
  3. 面向文档数据库
  4. 键值数据库

键值数据库

  • 数据以“键-值”的形式进行存储
  • 基本的数据处理速度非常快
  • 键值的数据库例子：redis，dynamo，riak等

图形数据库

  • 数据村委更为灵活的“文档”模式
  • 每个文档都可以右自己的结构
  • 面向文档数据库例子：MongDB，CouchDB，OrientDB等

面向文档数据库

  • 以“列”的形式进行存储
  • 基于列的数据例子：Cassandra，Hbase，Scylla等

图形数据库

  • 除了存储尸体数据，也存储各个实体间的关系
  • 非常适用于社交软件、物流系统
  • 基于列的数据库例子：Neo4j，ArangoDB等

MongDB基本使用

创建、删除数据库

  • 初始3个默认库：
    • admin：”root”数据库，被加入的用户将获得所有数据库权限，某些特定命令只能从admin数据库运行
    • config：MongDB内部专业，内容会随着版本变化而变化，随意修改会导致数据存储不稳定和不持续，用于分片设置
    • local：存储副本记得配置信息，只在本地存储数据
      初始3个默认数据库不要随意更改，存储和读取数据

创建流程

  • CREATE DATABASE 创建数据库
  • Database Name 数据库名字/Collection name 集合名字
    • 命名规则
    • 不可以包括特殊字符：/ \ . “ $ * < > : | ?
    • 已下划线或字母开头
    • 不可以有空格
    • 不可以是system.开头（内部专用）
    • 长度小于64字符
  • Create Database
