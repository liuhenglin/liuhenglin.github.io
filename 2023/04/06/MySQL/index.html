MySQL基础

数据模型

SQL通用语法

  1. SQL语句以分号结尾（；）
  2. SQL用空格和缩进增强可读
  3. 不区分大小写，但是建议关键字大写
  4. 注释：
  • 单行注释：–注释内容 或 #注释内容
  • 多行注释：/* 注释内容 */

SQL语句分类

DDL：数据定义语言，用来定义数据库对象（数据库，表，字段）一列一个字段
DML：数据操作语言，用来对数据库表中的数据进行增删改
DQL：数据查询语言，用来查询数据库中表的记录
DCL：数据控制语言，用来创建数据库用户、空值数据库的访问权限
cmd模式\c取消mysql语句输入

DDL

cmd mysql shell

库操作

说明 命令
查询所有数据库 show databases;
查询当前数据库 select database();
使用数据据库 use 数据库名;
查询当前数据库所有表 show tables;
查询表结构 desc 表名;
查询指定表的建表语句 show create table 表名;

创建数据库

数据库名 create database
create database[if not exists] 数据库名 [default charset 字符集] [collate 排序规则];
create database 数据库名；——创建数据库（不会重复创建）
create database if not exists 数据库名；——if如果有这个数据库就不创建，如果没有就创建
create database if not exists 数据库名 default charset 字符集 collate 排序规则
create database default charset utf8；设置为utf-8字符
一般设置字符的的推荐设置为utf8mb4，这个支持4个字节，utf8支持3个字节

删除数据库

drop database [if exists] 数据库名;
if exists 如果有就删除，没有就不删除

创建

create table 表名(
字段1 字段1类型[comment 字段1注释],
字段2 字段2类型[comment 字段2注释],
字段3 字段3类型[comment 字段3注释],
……
字段n 字段n类型[comment 字段n注释]
)[comment 表注释] （类型：数字型，浮点型，…）
例：
create table a(
id int comment ‘编号’,
name varchar(50) comment ‘姓名’,
age int comment ‘年龄’,
gender varchar(1) comment ‘性别’
)comment ‘用户表’;
mysql字符串用varchar(),括号里的是字符串最大长度

类型

mysql中主要分为三类：数值类型、字符串类型、日期时间类型

数值类型

类型 有符号范围(SIGNED) 无符号范围（UNSIGNED) 描述
TINYINT (-127，128) (0,255) 小整数值
SMALLINT (-32768,32767) (0,65535) 大整数值
MEDIUMINT (-8388608,8388607) (0,16777215) 大整数值
INT或INTEGER (-2147483648,2147483647) (0,4294967295) 大整数值
BIGINT (-2^63,2^63-1) (0,2^64-1) 极大整数值
FLOAT (-3.402823466 E+38,3.402823466351 E+38) 0和(1.175494351 E-38,3.402823466 E+38) 单精准浮点数值
DOUBLE (-1.7976931348623157 E+308,1.7976931348623157 E+308) 0和(2.2250738585072014 E-308,1.7976931348623157 E+308) 双精准浮点数值
DECIMAL 依赖于M(精度)和D(标度)的值 依赖于M(精度)和D(标度)的值 小数值(精确点数)
  • 精度是数字位数，标度是小数位数
  • 需要正数加上无符号范围（UNSIGNED）
  • 例年龄：age TINYINT UNSIGNED
  • DOUBLE需要设置参数（位数，小数位数）
  • 例：
    • 4位数，1位小数 位数=数字的长度包括小数位
    • DOUBLE(4,1)=100.0

字符串类型

类型 描述
CHAR 定长字符串
VARCHAR 变长字符串
TINYBLOB 不超过255个字符的二进制数据
TINYTEXT 短文本字符串
BLOB 二进制形式的长文本数据
TEXT 长文本数据
MEDIUMBLOB 二进制形式的中等长度文本数据
MEDIUMTEXT 中等长度文本数据
LONGBLOB 二进制形式的极大文本数据
LONGTEXT 极大文本数据
  • 二进制数据相当于视频，音频之类的
  • CHAR / VARCHAR后面需要跟参数，参数就是CHAR / VARCHAR最大长度可以有几个字符
  • 例：CHAR(10) = 最多有10位字符超过10个就是报错，即使只有1个字符也会用空格补齐其他的空位
    • VARCHAR(10) = 最多有10位字符超过10个就是报错，不会用空格补齐
    • CHAR：性能好
    • VARCHAR：性能较差

日期类型

类型 范围 格式 描述
DATE 1000-01-01至9999-12-31 YYYY-MM-DD 日期值
TIME -838:59:59至838：59：59 HH:MM:SS 时间值或持续时间
YEAR 1901至2155 YYYY 年份值
DATETIME 1000-01-01 00:00:00至9999-12-31 23:59:59 YYYY-MM-DD HH:MM:SS 混合日期和时间值
TIMESTAMP 1970-01-01 00:00:01至2038-01-19 03:14:07 YYYY-MM-DD HH:MM:SS 混合日期和时间值，时间戳

表修改

添加字段：
alter table 表名 ADD 字段名 类型(长度) [comment ‘注释’] [约束];
例：为b表增加一个新的字段“昵称”为nickname，类型为varchar(20)
alter table b add nickname varchar(20) comment ‘昵称’;
修改数据类型：
alter table 表名 modify 字段名 新数据类型(长度)；
修改字段名和字段类型：
alter table 表名 change 旧字段名 新字段名 类型(长度) [comment ‘注释’]；
例：将b表的nickname字段修改为username，类型为varchar(30)
alter table b change nickname username varchar(30) comment ‘用户名’;
删除字段：
alter table 表名 drop 字段名；
例:将b表的字段username删除
alter table b drop username;
修改表名：
alter table 表名 rename to 新表名;
例：将b表改为c表
alter table b rename to c;
表删除：
drop table [if exists] 表名;
if exists = 如果存在就执行，不存在就不执行，不存在不会报错
删除指定表，并重新创建该表：
truncate table 表名; = 删除数据保留结构
